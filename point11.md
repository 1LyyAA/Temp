# Пункт 11: Методы валидации данных

## Основные идеи:

1. **Holdout Method:**
   - Простое разделение на Training и Testing
   - Быстрый метод
   - Может быть неустойчивым при малых данных
   - Обычно 70-80% на обучение, 20-30% на тест

2. **Cross Validation (Кросс-валидация):**
   - Разделение на k частей (фолдов)
   - Каждая часть по очереди становится тестовой
   - Более надежная оценка качества
   - K-fold: обычно k=5 или k=10
   - Усреднение результатов по всем фолдам

3. **Training, Validation, Testing:**
   - Три независимых набора данных
   - Training: обучение модели
   - Validation: подбор гиперпараметров
   - Testing: финальная оценка
   - Используется при достаточном количестве данных

4. **Зачем нужна валидация:**
   - Оценка обобщающей способности
   - Избежание переобучения
   - Выбор гиперпараметров
   - Сравнение моделей
